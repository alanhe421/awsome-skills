# S1 信号解读

用于解释用户提供的 S1 链接或截图。S1 是小隐寺提供的量化辅助信号，主要针对美股训练，适用级别为日线；港股和 A 股并非其主要训练市场，解读时应降低置信度。信号可以辅助组织观察，但不能单独形成买卖建议。

定义与使用边界以 [S1 官方文档](https://xiaoyinsi.com/docs/s1-signal) 为准。涉及页面当前规则或定义时联网复核，并注明核验日期。

## 核心信号

- **B（Buy）**：潜在买点，不等于当天买入指令。当天信号可能消失，通常至少等待下一交易日确认。若下跌连续数字仍未中断，忽略单独出现的 B，等待下跌连续性结束或新的确认。
- **S（Sell）**：偏保守的卖点。优先进入减仓、止盈和风险控制判断，不把它当成普通震荡，也不默认由此建立空头仓位。若上涨连续阶段出现 S，可先保护利润，再观察次日 S 是否消失及上涨连续性是否延续。
- **数字 1–24**：描述上涨或下跌走势的连续性，不是独立买卖信号。重点判断方向、数字是否递增延续、何处中断，以及它与 B/S 的先后组合；中断只代表原连续行为可能变化，不保证立即反转，也可能转为震荡。
- **E / 橙色柱**：异常换手提示，后续波动风险可能放大，但方向不确定。
- **IB**：机构进货提示，也可能伴随诱多；次日可能出现较大波动，方向不能仅凭 IB 确定。

不要根据截图颜色臆测数字对应上涨还是下跌；应结合相邻价格走势和图例确认。看不清标记时明确写“无法识别”，不要猜测。

## 截图判读流程

按以下顺序读取，避免只盯住最新一个符号：

1. 先记录已知信息：ticker、交易所、图表周期、截图覆盖日期和价格区间。缺少 ticker、日线周期或完整价格区间时，说明局限，不给确定性结论。
2. 找出最近一次 B 或 S 的日期/相对位置，并说明它出现在上涨、下跌还是震荡背景中。
3. 验证信号：观察 B/S 后至少一个交易日的信号是否保留，以及价格走势是否支持该信号；截图止于信号当天时标记为“等待确认”。
4. 判断数字连续阶段：说明上涨或下跌连续性是否仍在延续、已经中断或因截图不全无法判断。数字大小本身不代表涨跌幅或信号强弱。
5. 检查组合：下跌连续数字未中断时弱化 B；上涨连续阶段出现 S 时优先风控，再看次日 S 和连续数字的变化。
6. 将当前状态归入且只归入一个主状态：**新买点候选**、**等待确认**、**持仓观察**、**减仓/止盈**。信息不足时使用“等待确认”，并列出需要补充的信息。

## 仓位情景框架

以下是条件式研究框架，不替用户决定仓位比例或价格：

### 空仓

- 没有新 B：不因连续数字本身追入，等待新信号或结构确认。
- 新 B 当天：列为“等待确认”，观察下一交易日 B 是否保留、下跌数字是否中断、价格是否停止创新低。
- 更完整的候选结构是“下跌后转为窄幅震荡—震荡中出现 B—未继续下跌—后续 B 经次日确认”；大幅上下波动不视为窄幅震荡。

### 已持仓且浮盈

- 无 S 且上涨连续性延续：归为“持仓观察”，同时用用户已有策略中的前低、支撑位、移动止损或最大回撤约束设置防守位；不要凭截图虚构具体价格。
- 出现 S：归为“减仓/止盈”，可讨论分批保护利润；次日若 S 消失且上涨连续性未中断，再重新评估剩余仓位，而不是撤销已做的风险管理解释。

### 已持仓且成本接近当前价

- B 未确认或下跌连续性仍在：避免把 B 直接解释为加仓依据，等待重新确认。
- S 出现或关键结构失守：优先讨论减少风险；若用户未提供成本、可承受损失和防守位，只说明需要这些信息，不给具体比例。

### 已持仓且浮亏

- 不以“摊低成本”为默认目标，也不因为单个 B 自动补仓。
- 先核对原投资逻辑、最大可承受损失、仓位集中度和流动性；下跌连续性未中断或出现 S 时，优先讨论风险暴露。
- 只有 B 经确认、下跌连续性中断且其他研究依据仍成立时，才把加仓列为一个待评估情景；仍需由用户结合自身约束决策。

## 输出模板

简洁回答也必须覆盖以下项目：

1. **当前信号状态**：从四个主状态中选择一个，并用一句话说明依据和置信限制。
2. **图上信号**：解释最近 B/S、数字连续性及其组合；出现 E/IB 时补充方向不确定的波动风险。
3. **验证情况**：信号已验证、被否定，还是需要下一交易日/更完整截图确认。
4. **按仓位讨论**：分别给空仓、持仓浮盈、成本附近、持仓浮亏的条件式框架；不知道用户仓位时不要擅自选一种代替用户。
5. **缺失信息**：列出 ticker、周期、日期、完整价格区间、持仓成本或风险约束中真正影响判断的项目。
6. **风险提示**：写明技术信号不能替代基本面、财报、估值、流动性和市场环境判断，并以“仅供研究教育，不构成投资建议”结尾。

不要把单个 B、S、数字、E 或 IB 写成确定性预测，不承诺收益，不输出未经用户信息支持的精确买卖价或仓位比例。
